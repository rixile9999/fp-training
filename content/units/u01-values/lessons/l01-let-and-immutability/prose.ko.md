<!-- segment: seg-01-let -->

## 값에 이름을 붙인다 — `let`

Gleam에서 `let`은 값을 담는 "상자"를 만드는 게 아니라, 이미 존재하는 값에
**이름표를 붙이는** 일입니다.

```gleam
let score = 42
let doubled = score * 2
```

이름표를 붙인 뒤에 값을 "수정"하는 방법은 없습니다 — Gleam의 모든 값은
불변(immutable)입니다. 값을 바꾸고 싶다면 **새 값을 만들어 새 이름에
붙입니다**. 이 단순한 규칙이 이후 record update(U4), 누산기(U6)까지
이어지는 핵심 사고방식입니다.

<!-- segment: seg-02-shadowing -->

## 수정이 아니라 가리기 — shadowing

같은 이름으로 다시 `let` 하는 것은 합법입니다. 하지만 이것은 값을 바꾸는 게
아니라, **새 바인딩이 이전 것을 가리는(shadowing)** 것입니다.

```gleam
let x = 1
let x = x + 10   // 새 x = 11. 옛 x(= 1)는 바뀐 게 아니라 가려졌을 뿐
```

값이 바뀐 게 아니라 이름이 새 값을 가리킬 뿐이므로, **이전 값을 참조하던
코드는 영향받지 않습니다**. 반면 `let` 없는 재대입(`x = x + 10`)은 컴파일
에러입니다 — Gleam에 재대입 연산자는 없습니다.
